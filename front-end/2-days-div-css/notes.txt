2、块级元素和内联元素：
	块级元素：一般是其他元素的容器，科容纳内联元素和其他块级元素，块级元素排斥其他元素与其位于同一行，宽度和高度起作用；
	内联元素：只能容纳文本和其他内联元素，允许其他内联元素与其位于同一行，宽度和高度不起作用。

3、IE6 双边距 BUG：
	要为块级元素；
	要左侧浮动；
	要有左外边距[margin-left]。

	解决办法：在块级元素加上属性：display: inline;

9、定位[position]：
	不设定 position 属性时，元素的默认属性是 static;
	绝对定位：默认参照浏览器的左上角，配合 TOP、RIGHT、BOTTOM、LEFT 定位；

	绝对定位的属性：
		如果没有TRBL，以父级的左上角，在没有父级的时候，他是参照浏览器左上角，如果在没有父级元素的情况下，存在文本，则以他前面的最后一个文字的右上角为原点进行定位但是不断开文字，覆盖于上方；
		如果设定TRBL，并且父级没有设定position属性，那么当前的absolute则以浏览器左上角为原始点进行定位，位置将由TRBL 决定；
		如果讴定 TRBL，幵丏父级讴定 position 属性(无论是 absolute 还是 relative)，则以父级的左上视为原点进行定位，位置由 TRBL 决定。即使父级有 Padding 属性，对其也不起作用，说简单点就是：它只坚持一点，就以父级左上角为原点进行定位，父级的 padding 对其根本没有影响。

	要把一个定位属性为 absolute 的元素定位于其父级元素内，必须满足条件：[只要有一点不满足，元素就会以浏览器左上角为原点]
		设定 TRBL
		父级设定 position 属性。

	相对定位：默认参照父级的原始点为原始点，配合 TRBL 进行定位，当父级内有padding 等 CSS 属性时，当前级的原始点则参照父级内容区的原始点进行定位。

	相对定位的属性：
		如果没有 TRBL，以父级的左上角，在没有父级的时候，他是参照浏览器左上角(到这里和 absolute 第一条一样)，如果在没有父级元素的情况下，存在文本，则以文本的底部为原始点进行定位并将文字断开(和 absolute 不同)。
		如果讴定 TRBL，幵丏父级没有讴定 position 属性，仌旧以父级癿左上觇为原点迕行定位(和 absolute 丌同)
		如果讴定 TRBL，幵丏父级讴定 position 属性(无讳是 absolute 迓是 relative)，则以父级癿左上觇为原点迕行定位，位置由 TRBL 决定(前半段和 absolute 一样)。如果父级有 Padding 属性，那举就以内容区域癿左上觇为原点，迕行定位(后半段和absolute 丌同)。

	[相对定位]总结：无论父级元素是否存在，无论有没有TRBL，均是以父级的左上角进行定位，但是父级的padding属性会对其影响。

	采用定位来布局页面的总结：如果用定位来布局页面，父级元素的 position 属性必须为 relative，而定位于父级内部某个位置的元素，最好用 absolute，因为它不受父级元素的 padding 的属性影响，当然也可以用 relative，计算的时候需要加上父元素的 padding 值。

10、CSS Hack
	一种兼容 CSS 在不同浏览器中正确显示的技巧方法。
	!important: IE6 不能识别；
		#content { height: 960px !important; /* for IE7/8/9 */ height: 900px; /* for IE6 */ }
	*：IE8/IE9/FireFox 不能识别
		#content { height: 960px; *height: 900px; /* for IE6/IE7 */}

11、text-indent: 2em; [缩进2个文字]。
	text-indent 的意思表示文字缩进；
	单位 em 用来表示文字的高度，2em 就是两个文字的高度。但是由于汉字的宽度和高度一样，所以可以用来表示缩进两个文字的宽度。
	注意：此属性只能加载块级元素上面，对内联元素是不起作用的。
