W3C标准：由万维网联盟指定的一系列标准，包括：
  • 结构化标准语言（HTML和XML）
  • 表现标准语言（CSS）
  • 行为标准语言（DOM和ECMAScript）
    • 倡导结构、样式、行为分离
    CSS中，存在3中的定位机制
  • 标准文档流（Normal flow）
  • 浮动（Floats）
  • 绝对定位（Absolute positioning）
    标准文档流
特点：从上到下，从左到右，输出文档内容 由块级元素和行级元素组成； 块级元素：从左到右撑满页面，独占一行，触碰到页面边缘时，会自动换行 div ul li dl dt p ... 行级元素：能在同一行内显示，不会改变HTML文档的结构 span strong img input 块级元素和行级元素都是盒子模型 盒子模型是网页布局的基石 边框 border 外边距 margin 内边距 padding 内容 content 三个值：上 [左和右] 下 两个值：[上和下] [左和右] 一个值：[上右下左] 盒子3D模型： border content+padding background-image background-color margin 盒子的尺寸：边框+外边距+内边距+盒子中内容的尺寸 自动居中一列布局： 三个技能点： 标准文档流； 块级元素； margin属性。 margin: 0 auto; auto 会根据浏览器的宽度自动的设置两边的外边距； 原理：(浏览器宽度 - 外包含层的宽度) / 2 = 外边距 如果想让页面自动居中，当设置margin属性为auto时，不能再设置浮动或者绝对定位属性。 浮动布局： CSS中规定的第二种定位机制： 能够实现横向多列布局； 通过设置float属性实现； float: left; float: right; float: none; 特点：元素会左移或右移，直至触碰到容器为止。 注意：设置了浮动的元素，仍然处于标准文档流中 当元素没有设置宽度值，而设置了浮动属性，元素的宽度随内容的变化而变化； 当元素设置浮动属性后，会对相邻元素产生影响，相邻元素特指紧邻后面的元素； 清除浮动的常用方法： 设置clear属性：clear: both; clear: left; clear: right; 同时设置宽度width: 100%(或固定宽度) + overflow: hidden; 添加空标签
，不推荐。
